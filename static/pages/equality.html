Javascript by Example: Equality

JS supports three different value comparison equality operations, but we will only focus on two of them as they are more widely used: strict equality comparison (===, AKA "identity", "tripe equals") and abstract equality comparsion (==, AKA "loose equality", "double equals").

Loose equality (==) compares two values after performing a type conversion to the values being compared. A full list of the conversions can be found here. Common cases are shown in this example:

(a) string representing a number value when compared to the number itself evaluates to true.

(b) Values that are considered falsy in JS that are compared using == evaluates to true. Falsy values in JS are values that are considered false in the context of booleans. In JS, these values include: false, 0, -0, 0n, "", null, undefined, and NaN. The exception to this rule is NaN, which compared to any falsy value, including NaN, returns false

(c) An array compared to a string that is equal to the values in the array separated by a comma(",") evalues to true.

Strict equality (===) compares two values without performing any type conversion. Strict equality evalutes to true if the values have the same type, are not numbers and have the same value OR if both values are numbers, then only if they are both not NaN, and are the same value or if one is +0 and the other is -0.

When comparing two objects of the same structure, both loose equality and strict equality will return false, unless you are comparing an object to itself.

In most cases, using loose equality is highly discouraged due to predictability of type conversion.
