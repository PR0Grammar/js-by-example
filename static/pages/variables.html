Javascript by Example: Variables

There are three keywords associated with variables in JS: const, let and var.

All variable types are hoisted in JS. JS hoisting for variables is the process of allocating memory prior to execution of the code.

In pre-2015 JS, var was the only available keyword for defining variables. Variables defined by var are scoped to the immediate function body and are intialized with the value undefined during hoisting. Variables defined with var outside of a function becomes part of the global scope.

let and const were introduced in the ES6 standard. Unlike var, they are scoped to the immediate block that they are declared in, not the immediate function. Also unlike var, let and const cannot be read or assigned until program control passes the point of declaration. The interim moment is known as the temporal dead zone (TDZ) - the variables are in scope, but not yet declared. If there is an attempt to access the variable before declartion, JS will throw a ReferenceError.

let and const are almost exactly the same - the only difference is let variables can be reassigned while const variables cannot be reassigned.
